Belltower — School Onboarding Guide

Step-by-step setup for a new school from scratch. Follow in order.

Internal — Admin use only
Keep updated as platform changes
Before You Start — Gather These
  • School's full name and short name (abbreviation for tight UI spaces)
  • Staff email domain — the part after @ in their Google accounts (e.g. lincoln.edu). Must be exact.
  • Timezone (e.g. America/New_York)
  • Complete grade list the school uses, in order, including the terminal/graduating grade
  • Whether school uses homeroom teacher assignments
  • Staff roster spreadsheet: first name, last name, email, position
  • Student/family roster: student names, grades, family names, carline tag numbers
  • Which Belltower modules they need (Leave, Carline, Compliance, Licensure, Field Trips, Substitutes)
  • If using carline: bus group names and which students ride each bus
  • If using Leave: workday hours (e.g. 8), annual leave allotment per employee type
⚠

Carline tag numbers: Every family needs one. These are the physical numbered tags families hang in their car. Collect the full list before upload — you cannot bulk-load families without them. Numbers must be unique across the school.

Phase 1 — Create the School Record
1
Create the school
Admin Panel → Settings → Schools (superadmin only)

Click Add School. Fill in:

  • Name — full school name
  • Short name — abbreviation used in tight UI spaces
  • Email domain — staff Google account domain, e.g. lincoln.edu
  • Timezone — e.g. America/New_York. Affects leave calendar and email notification times.
  • Weather lat/lon — optional, enables weather widget on admin dashboard
⚠

Email domain is critical. When a staff member signs in with Google for the first time, Belltower checks their email domain against this field to auto-link them to the school. If the domain doesn't match (e.g. you entered lincoln.edu but their email is staff.lincoln.edu), their profile will be created as a floating pending account with no school attachment.

2
Configure grade levels
Admin Panel → Settings → Schools → Edit school

This is the single most important configuration step. Enter the exact grade level strings the school uses, in ascending order. Examples:

  • Elementary: PK, K, 1, 2, 3, 4, 5
  • K–8: K, 1, 2 … 8

Also set: Terminal grade (the last grade before students graduate/leave — used by Year-End Promotion to automatically mark students inactive).

Set Uses homerooms if teachers are assigned as homeroom teachers. Set Require MVR for drivers if the school requires a motor vehicle record check for chaperones who drive.

⚠

Grade level strings must be exactly consistent everywhere. Student records, bulk upload sheets, class placement, carline filtering, and year-end promotion all match on these exact strings. If you enter Pre-K here but the bulk upload sheet says PreK, those students will have mismatched grades that won't filter correctly. Decide on the format before touching anything else.

3
Enable modules
Admin Panel → Settings → Schools → Edit school → Module toggles

Toggle on only the modules this school is using. Unused modules clutter the nav for their staff. You can enable more later. Modules available:

  • Leave — leave requests, approvals, balances, substitutes trigger
  • Substitutes — substitute assignment board
  • Carline — dismissal screen, carline input, bus groups, carpools in nav
  • Licensure — staff license tracking and expiry alerts
  • Compliance — background checks and volunteer agreement forms
  • Field Trips — trip planning, chaperone management, vehicle assignment
4
Set up campus(es) — only if using Carline
Admin Panel → Directory → Campuses

Standalone single-campus schools do not need a campus record unless they are using the Carline module. If carline is enabled, create at least one campus — the carline screen uses campus to filter students. Name it something clear (e.g. "Main Campus" or the campus name).

ℹ

Multi-campus schools should set up all campuses before uploading students, since students can be assigned to a campus during bulk upload.

Phase 2 — Load Data
5
Bulk upload employees (staff)
Admin Panel → Settings → Bulk Upload

Download the Bulk Upload Template. Fill in the Employees sheet. Required columns: first name, last name, email, position. Upload → Preview → Commit.

⚠

Employee emails must exactly match their Google sign-in email. This is what Belltower uses to auto-link their login to their employee record. A wrong email means they'll create a floating pending profile on first login that you'll need to manually resolve.

ℹ

Uploading creates the employee record but does not create a login. Staff still need to sign in with Google themselves first (see Step 8).

If something looks wrong after Commit, use Undo Last Upload immediately — it rolls back the entire batch cleanly.

6
Bulk upload students and families
Admin Panel → Settings → Bulk Upload

Fill in the Students and Families sheets. Key fields:

  • Family: carline_tag_number — required, must be unique per school, must not be blank. Use the physical tag numbers families already have (or assign new ones).
  • Student: grade_level — must match the exact strings configured in Step 2.
  • Student: family_name — must match the family name on the Families sheet so the link is created correctly.
  • Student: campus — if using multi-campus, match the campus name from Step 4 exactly.
⚠

Duplicate carline tag numbers will cause the upload to fail. Every family must have a unique tag. Check the sheet for duplicates before uploading.

⚠

Grade level mismatch. If a student's grade_level string doesn't match the school's configured grade levels, they will exist in the system but won't appear correctly in carline filtering, class placement, or year-end promotion.

7
Set up bus groups — only if using Carline
Admin Panel → Directory → Bus Groups

Create a bus group for each bus/route (e.g. "Bus 1", "Route A"). Then assign students to their bus group — either from each student's record or via the bus group detail panel. Bus group buttons appear on the carline screen and call all assigned students at once.

ℹ

Students not assigned to any bus group only appear via family tag lookup. Bus groups are separate from and additive to the family tag system.

Phase 3 — Set Up User Accounts
8
Activate the first admin account
Admin Panel → Settings → User Access & Permissions → Pending Users

The school's designated admin needs to sign in with Google first before you can activate them. After they sign in, their account appears in the Pending Users table at the bottom of the Access page.

  1. Have the school admin go to belltower.school and sign in with Google.
  2. In the Admin Panel, go to User Access & Permissions.
  3. Scroll to Pending Users and click Activate next to their name.
  4. Their account status changes to Active and their login is linked to their employee record.
  5. Now grant them permissions — see Phase 4 and the Permissions Reference at the end of this document.
⚠

Staff must sign in before you can activate them. You cannot pre-activate an account. The pending record only appears after their first Google sign-in.

⚠

If their pending account shows no school attached (blank school name in the pending table), the email domain didn't match. You'll need to manually link them in Supabase: find their profile row, set the correct school_id, and update status to active.

9
Grant permissions to the school admin
Admin Panel → Settings → User Access & Permissions

Select the newly activated admin. Use the Role Presets dropdown → Admin to apply a full permission set, then adjust as needed. At minimum, a school admin needs:

  • can_access_admin — access to the Admin Panel at all
  • can_manage_access — so they can activate other staff accounts themselves
  • can_manage_staff, can_manage_students, can_manage_families
  • can_bulk_upload, can_export_data
  • Appropriate module permissions based on enabled modules
⚠

can_manage_access is powerful. A user with this permission can give themselves or others any permission, including can_manage_access itself. Only grant it to fully trusted admins.

10
Hand off remaining account activations to the school admin

Once the school admin can access the Access & Permissions panel, they can activate all remaining staff themselves. Share these instructions with them:

  1. Have each staff member sign in at belltower.school with their school Google account.
  2. The admin goes to Admin Panel → User Access & Permissions → Pending Users.
  3. Click Activate for each person, then set their permissions using the Role Presets.
Phase 4 — Configure Active Modules
11
Leave setup — only if Leave module is enabled
Leave Management → Adjust Balances → Annual Allotments

Before staff can use leave, allotments must be applied:

  1. Go to Leave Management (staff portal, admin view).
  2. Under Adjust Balances, find the Annual Allotments section.
  3. Set the allotment hours per leave type for each employee (or run bulk allotment for all staff at once).
⚠

If allotments are skipped, staff see a $0 balance and think something is broken. This is the most common source of confusion on day one. Do this before staff start logging in.

Also verify leave types are configured correctly — which types count against balance vs. tracking only (Jury Duty, Bereavement, Professional are typically tracking-only).

12
Compliance setup — only if Compliance module is enabled
Admin Panel → Operations → Compliance → Form Templates

Background check tracking works out of the box — staff can start entering records immediately. For volunteer agreement forms:

  1. Go to Compliance → Form Templates.
  2. Click New Template. Enter the form name, upload the school logo, and paste the agreement text.
  3. Save the template. It will generate a shareable link.
  4. Activate the form link so guardians can access it.
ℹ

The school logo must be uploaded to Supabase Storage (bucket: school-assets) before it will appear on the volunteer form. This is done through the school settings panel.

13
Verify setup with the Setup Checklist
setup.html (accessible from Admin Panel → Settings → Setup Checklist)

Open the Setup Checklist page, select this school (superadmin), and confirm every step shows a green checkmark. Pay attention to:

  • Any red ✗ — these are blocking issues that will cause features to not work
  • Any amber ⚠ — these are gaps that will cause confusion or degraded functionality

Share the setup checklist link with the school admin so they can monitor their own progress.

Permissions Reference

All permissions are toggles on the profiles table. The Admin Panel → User Access & Permissions panel manages these. Role Presets apply a predefined bundle.

ℹ

Permissions are layered. can_access_admin is the door key — it lets the user into the Admin Panel at all. The granular permissions below are the room keys — they control which tabs and sections appear once inside. A user with only can_access_admin and no other permissions will see an essentially empty admin panel with just the Setup Checklist and Requests form management nav links.

can_manage_studentsStudents tab — add, edit, manage student records.
can_manage_familiesFamilies tab + Data Collection (Guardian Intake campaigns).
can_manage_guardiansGuardians tab + Data Collection tab.

can_manage_bus_groupsBus Groups tab.
can_manage_carpoolsCarpools tab.
can_manage_campusesCampuses tab.
can_manage_calendarAdd and edit events on the dashboard School Calendar. Grantable to front office, not just admins.

can_review_ptoLeave pending requests tab — see requests awaiting decision.

Module Gates — What Each Module Unlocks

Modules are toggled in Schools → Edit school. A module must be enabled AND the user must have the matching permission for nav links to appear.

Module keyNav items unlockedRequired permission(s)
pto Leave Management (admin), Leave (staff portal) can_view_pto_calendar, can_review_pto, can_approve_pto, or can_generate_pto_reports
substitutes Substitutes (admin + staff portal) can_manage_substitutes
carline Carline screen, Carline Input, Bus Groups, Carpools can_view_carline or can_manage_carline (for carline); can_manage_bus_groups, can_manage_carpools (for admin tabs)
licensure Licensure tab (admin), license alerts on staff portal dashboard can_manage_licensure
compliance Compliance tab (admin + staff portal link), chaperone compliance on field trips can_manage_compliance
field_trips Field Trips (staff portal nav) None — all staff see field trips they are managing teachers on. can_manage_field_trips grants admin view of all trips.
requests Requests form management (admin panel nav), Submit a Request (staff portal nav) Admin form management: can_access_admin. Staff submission: none — any authenticated staff member can submit. Defaults to enabled (no explicit school toggle required).
ℹ

Class Placement is permission-gated, not module-gated. The Class Placement tool (/app/placement.html) is always available as a standalone page — no module toggle required. It shows in the Staff Portal nav and Admin Panel nav for any user with can_manage_placement. Enable this permission for teachers during placement season, then remove it once placements are committed.

Common Pitfalls & How to Fix Them
SymptomRoot causeFix
Staff member signs in but sees "account pending" or gets routed wrong Email domain doesn't match schools.email_domain, so their profile was created with no school In Supabase table editor: find their row in profiles, set correct school_id, set status = 'active', can_login = true
Students don't appear in carline when tag is entered Student's grade_level string doesn't match the campus filter, or student has no family linked, or family tag number doesn't match Check student record for correct grade_level and family_id. Check family's carline_tag_number matches what was entered.
Bulk upload fails on families Duplicate carline_tag_number in the sheet, or a tag number left blank Find and fix duplicates/blanks in the spreadsheet. Every family must have a unique non-empty tag.
Students appear in system but wrong grade filtering in carline / placement grade_level string in student record doesn't match school's configured grade_levels Fix grade_level on the student record to match exactly. Can bulk-fix in Supabase table editor with a SQL UPDATE if widespread.
Staff see $0 leave balance on day one Annual allotments were never run Leave Management → Adjust Balances → Annual Allotments → run for all staff
Admin nav link doesn't appear for a user can_access_admin is false on their profile, or the module is disabled, or the specific granular permission is missing Go to User Access & Permissions, find the user. If they can't reach the admin panel at all, toggle on can_access_admin. If they're in the admin panel but a specific tab is missing, check the matching granular permission (e.g. can_manage_staff for the Staff tab).
Staff member can't see "Submit a Request" in their portal The requests module is disabled for the school Admin Panel → Schools → Edit school → enable the Requests module toggle. Note: all authenticated staff can submit once the module is on — no per-user permission needed.
Carline screen shows "no students" even with families loaded No campus record exists (required when carline module is on), or wrong campus selected Add a campus in Admin → Campuses. Ensure students are assigned to that campus if multi-campus.
A staff member's profile is linked to the wrong school Email domain matched a different school's record, or they logged in before the school was set up In Supabase profiles table: update school_id to the correct school UUID
Leave calendar times are wrong (off by hours) Timezone not set on the school record Edit school in Schools tab, set the correct IANA timezone string (e.g. America/Chicago)
Year-end promotion graduates wrong students or misses students Terminal grade not set, or students have blank/mismatched grade_level Set terminal_grade on the school. Fix any students with blank or mismatched grade_level before running promotion.
Teacher doesn't see Class Placement link in their portal can_manage_placement is not enabled on their profile Admin Panel → User Access & Permissions → find the staff member → toggle on can_manage_placement. The link appears in their Staff Portal nav immediately on next page load.
Supabase Direct Access — Emergency Fixes

For issues that can't be resolved through the UI. Use the Supabase Table Editor or SQL Editor. Always make a note of what you changed and why.

Key tables for manual fixes
schoolsSchool config, grade_levels, timezone, modules
school_modulesEnable/disable modules per school
profilesUser accounts, permissions, school linkage, status
employeesStaff records, email, position, active flag
studentsStudent records, grade_level, family_id, campus_id
familiesFamily records, carline_tag_number (must be unique)
pto_balancesCurrent leave balance per employee per type
campusesCampus records — required for carline
Useful SQL snippets
Find a user's profile by email SELECT * FROM profiles WHERE email = 'x@school.edu';
Fix a profile's school UPDATE profiles SET school_id = '<uuid>' WHERE email = 'x@school.edu';
Activate a pending profile UPDATE profiles SET status = 'active', can_login = true WHERE email = 'x@school.edu';
Check which modules are on for a school SELECT * FROM school_modules WHERE school_id = '<uuid>' AND enabled = true;
Find students with mismatched grade levels SELECT grade_level, count(*) FROM students WHERE school_id = '<uuid>' GROUP BY grade_level;
ℹ

RLS is enabled on all tables. SQL run in the Supabase SQL Editor uses the postgres role and bypasses RLS — changes take effect immediately regardless of policy. Be careful. Always WHERE-clause by school_id so you don't accidentally modify another school's data.

Quick-Reference Order Checklist
  • Gathered all school info (domain, timezone, grades, roster, tag numbers)
  • Created school record in Admin Panel → Schools
  • Configured grade levels (exact strings) and terminal grade
  • Enabled correct modules
  • Created campus record (only if carline is enabled)
  • Bulk uploaded employees — verified emails match Google accounts
  • Bulk uploaded students and families — verified tag numbers unique, grade levels match
  • Set up bus groups and assigned students (only if carline is enabled)
  • School admin signed in with Google (creates pending account)
  • Activated school admin account and granted full permissions
  • Applied leave allotments for all staff (only if Leave is enabled)
  • Created compliance form templates (only if Compliance is enabled)
  • Ran Setup Checklist — all items green
  • School admin can log in and see correct data
  • Handed off remaining staff activations to school admin
